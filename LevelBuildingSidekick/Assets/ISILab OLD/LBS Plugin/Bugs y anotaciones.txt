
### Bugs ###

- Las ventanas pueden perder referencias no acesibles por los desarrolladores provocando que estan
  no se muestren.
	- Una solucion a esto puede ser reiniaar unity, o reescalar la ventana en cuestion.

- Aveces al hacer cambios en el codigo y tener una ventana abierta esta empieza arojar null refs
  esto ocurre apesar de que las variables que y al linea de codigo que dice tener null references
  tiene a todas las variables implicadas con valores correctamente asignado (se comporobo).
	- Esto ocurre cuando se tiene abierta las ventnas y el debbug de visualstudio activado, se
	  soluciona refrescando la ventana vuelve a aparecer al editar algun codigo.
	- Tambien courrio al momento de subir cmabios al git.

- El uso de ClassDropdown necesita que las clases a la cual hace referencia no este dentro de un nemespace
  ya que el estar en el namespace este no las puede encontrar a traves de reflexion por lo que termina
  arrojando un null reference.
	- Una solucion temporar para esto es dejar las clases que se necesitan para este elemento fuera de un namespace
	  apesar de que esto implique mas desorden en el codigo.
